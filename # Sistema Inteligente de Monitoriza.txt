---

# Contexto do Projeto

A aplicação tem como objetivo ajudar pessoas com diabetes a monitorizar os seus níveis de glicemia.

A ideia principal consiste em permitir que os valores sejam registados automaticamente através de um ESP32.

No entanto, essa comunicação será apenas simulada para fins académicos.

Não deve existir qualquer integração com dispositivos médicos reais.

Não devem ser utilizados protocolos proprietários nem APIs de fabricantes.

Toda a receção de dados deve ser simulada internamente.

Durante a apresentação da PAP será explicado que esta funcionalidade representa apenas uma prova de conceito.

# Simulação do ESP32

A aplicação não deve comunicar com qualquer equipamento real.

Criem um endpoint específico na API responsável por simular o envio de uma nova medição.

Exemplo do fluxo:

1. O utilizador pressiona o botão "Gerar Medição Automática (Simulação ESP32)".

2. O Frontend envia um pedido POST para a API.

3. A API gera uma medição simulada.

4. A medição é guardada na base de dados.

5. A IA analisa automaticamente o novo valor.

6. Caso seja necessário, são gerados alertas.

7. O Dashboard é atualizado automaticamente.

Todo este fluxo deve parecer exatamente igual ao funcionamento de um sistema real.

# INTELIGÊNCIA ARTIFICIAL

A IA deverá funcionar como um assistente educativo.

Nunca deverá substituir um médico.

Nunca deverá realizar diagnósticos.

Nunca deverá recomendar medicamentos.

Nunca deverá alterar doses de insulina.

Sempre que gerar uma resposta deverá apresentar um aviso semelhante a:

"Esta análise é apenas informativa e não substitui a avaliação de um profissional de saúde."

A IA deverá analisar automaticamente:

• Tendência da glicemia
• Evolução semanal
• Evolução mensal
• Frequência de valores elevados
• Frequência de valores baixos
• Horários com maior incidência
• Dias da semana com mais alterações

Depois deverá gerar:

Resumo diário

Resumo semanal

Resumo mensal

Recomendações educativas.

Exemplos:

• Tem registado valores elevados durante a manhã.
• Existe uma melhoria relativamente à semana anterior.
• Foram identificados vários episódios de hipoglicemia.
• Recomenda-se contactar o médico caso esta tendência continue.

As respostas devem ser claras.

Naturais.

Sem linguagem demasiado técnica.

# API REST

A aplicação deve expor uma API REST completa.

# PADRÕES DA API

Utilizar:

• RESTful principles
• JSON como formato padrão
• HTTP status codes corretos

Métodos:

• GET → leitura
• POST → criação
• PUT → atualização total
• PATCH → atualização parcial
• DELETE → remoção

---

# FORMATO DE RESPOSTAS

Todas as respostas da API devem seguir um formato consistente:

Exemplo:

{
"success": true,
"data": {},
"message": "",
"errors": []
}

Em caso de erro:

{
"success": false,
"data": null,
"message": "Erro",
"errors": []
}
